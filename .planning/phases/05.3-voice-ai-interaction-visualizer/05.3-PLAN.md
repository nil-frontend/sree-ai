# Phase Plan 05.3: Voice AI Interaction & Visualizer

## Overview
Implement a high-fidelity voice interaction experience featuring real-time audio visualization and a seamless "Speech-to-Text-to-Speech" loop.

## 🛠 Step 1: Real-time Audio Visualization
Create a reactive UI that pulses and scales based on microphone input levels.

- [ ] **Create `VoiceVisualizer.tsx`**:
    - Use Web Audio API (`AudioContext`, `AnalyserNode`).
    - Extract frequency data or RMS (volume) level.
    - Render a SVG or Canvas based visualization (e.g., concentric circles pulsing).
- [ ] **Integrate into `VoicePage.tsx`**:
    - Pass the `MediaStream` from `startRecording` to the visualizer.

## 🛠 Step 2: Backend TTS Integration
Enable the AI to "talk back" by generating audio from text.

- [ ] **Update `backend/src/services/ai.service.ts`**:
    - Add `generateSpeech` method using an NVIDIA NIM TTS model (e.g., `nvidia/parakeet-ctc-1.1b-large` or relevant).
- [ ] **Add Controller Endpoint**:
    - `POST /api/ai/tts` to accept text and return audio data (or a link).

## 🛠 Step 3: The Interaction Loop
Handle the full voice-driven conversation flow.

- [ ] **Update `VoicePage.tsx` logic**:
    - **Step A**: Record & Transcribe (Existing).
    - **Step B**: After transcription, send text to chat API for a response.
    - **Step C**: Send AI response to TTS API.
    - **Step D**: Play the returned audio immediately.

## 🛠 Step 4: UI/UX Refinement
- [ ] Add "Stop Talking" button to interrupt AI speech.
- [ ] Implement a smooth "Thinking" state transition between transcription and speech playback.

## 🛠 Step 5: Verification
- [ ] Test recording in different noise levels.
- [ ] Verify latency of the Speech-Chat-TTS loop.
- [ ] Ensure the visualizer is performant (uses `requestAnimationFrame`).
