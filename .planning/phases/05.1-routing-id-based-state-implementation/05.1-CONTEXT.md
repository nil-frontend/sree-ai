# Phase 05.1: Routing & ID-based State - CONTEXT

## Domain
ID-Based session management and application-wide routing overhaul.

## Prior Decisions
- **Lazy Session Creation**: Conversations are created at `/chat` (no ID) and only assigned a UUID + redirected to `/chat/:id` after the first user message.
- **Routing Overhaul**: Remove the `/dashboard` prefix from all routes for a cleaner URL structure (`/chat`, `/voice`, etc.).
- **Voice Handoff**: Post-voice sessions land on `/voice/chat/:id`. Transcripts must be italicized (`font-style: italic`).
- **Sidebar Context**: The Sidebar component will dynamically filter based on the current context (`chat` vs `voice`) and change "New Chat" to "New Conversation" in Voice mode.

## Specifics
- **Voice Page UI**: Must include a "Neural Activity" animation (vibrant pulsing elements) that reacts to real-time audio input (slated for 5.3, but 5.1 must establish the route).
- **Session Types**: Strict segregation in `conversations.type` between `chat` and `voice`.
