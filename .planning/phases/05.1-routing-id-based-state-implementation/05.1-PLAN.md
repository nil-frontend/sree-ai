# Phase Plan 05.1: Routing & ID-based State Implementation

## Overview
Overhaul the application routing to remove the `/dashboard` prefix and implement unique session identifiers in the URL for both Chat and Voice transcripts.

## 🛠 Step 1: Global Routing Update
Remove the `/dashboard` namespace and establish new route patterns.

- [ ] **Modify `frontend/src/App.tsx`**:
    - Update all `<Route>` tags to remove `/dashboard`.
    - Rename `/dashboard/voice` to `/voice`.
    - Add path pattern: `/chat/:id?` for text chats.
    - Add path pattern: `/voice/chat/:id?` for voice handoffs.
- [ ] **Update Navigation Components**:
    - Search and replace `/dashboard` links in `Navbar.tsx`, `Sidebar.tsx`, and any redirect logic.

## 🛠 Step 2: Session Persistence Logic (Zustand)
Update the store to handle deep-linking and on-demand data fetching.

- [ ] **Refactor `useChatStore`**:
    - Enhance `setActiveConversation` to perform a Supabase `fetch` if the ID isn't found in its current local list.
    - Add `fetchMessageById` helper to support deep-linking from shared URLs.

## 🛠 Step 3: Lazy Session Creation (ChatPage)
Implement the "start clean, id later" workflow.

- [ ] **Update `ChatPage.tsx`**:
    - Inject `useParams` to read `:id`.
    - **Initial State**: If no ID, show a clean message input.
    - **Creation Hook**: Upon the first message, call `createConversation`, retrieve the new ID, and use `Navigate` to update the URL to `/chat/:id`.
    - **Context Awareness**: Apply `font-style: italic` to message bubbles if the conversation `type` is `voice`.

## 🛠 Step 4: Component Integration & Polish
Ensure all UI triggers correctly respect the new routing.

- [ ] **Sidebar "New Chat" Refactor**:
    - Update the button to simply navigate to `/chat` (standard) or `/voice` (initiation).
- [ ] **Redirect Verification**:
    - Ensure root `/` and catch-all `*` correctly point to `/chat` (or `/voice` as default dashboard).

## 🛠 Step 5: Regression Testing
- [ ] Verify Settings, Image Generation, and Profile still function under unprefixed routes.
- [ ] Verify that refreshing a `/chat/:id` page correctly reloads messages.
